# Project rules

## Plan files are living documents

Whenever working from a plan file (default: [docs/plan.md](docs/plan.md), and any future plan files we create), keep it in sync with reality:

- When a todo derived from the plan moves to `completed`, mark the corresponding step as `✅ DONE` in the plan file in the same turn.
- If the work produced artifacts the plan didn't explicitly list (e.g. an extra preview file, a doc), add them under "Critical files" or as a sub-bullet of the relevant step.
- If the approach diverged from what the plan said, update the plan to describe what we actually did — not what was originally intended.

The plan file is the canonical source of "what's done, what's left." Treat it like a README that gets updated, not a snapshot.
